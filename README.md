# QTerrainSim — Quantum Similarity Learning for Mars Terrain Retrieval

**Paper:** *Quantum Similarity Learning for Mars Terrain Retrieval via Entangled-Pair Encoding*
**Venue:** IEEE Transactions on Geoscience and Remote Sensing (submission draft)
**Dataset:** HiRISE Mars v3 — 73,031 patches, 8 terrain classes
**Stack:** PyTorch 2.1 · PennyLane 0.45 · Python 3.10+

---

## Overview

QTerrainSim frames Mars terrain analysis as a **retrieval problem** rather than classification: given a query terrain patch, rank the archive by visual similarity. The core contribution is an *entangled-pair Parametric Quantum Circuit* (PQC) used as a Siamese similarity head.

The architecture encodes two CNN embeddings via interleaved `RX(π·eₐ[i])` / `RY(π·e_b[i])` rotations on the same qubits — both embeddings condition each wire's state before any entangling gate. This contrasts with the sequential axis encoding in SLIQ (AAAI 2023), where the first encoding block is effectively a no-op because `RZ|0⟩ = |0⟩` up to a global phase.

Barren plateaus are suppressed by using local Pauli-Z measurements on each wire (Cerezo et al., *Nature Commun.* 12:1791, 2021). Hard negative mining targets the two geologically confusable terrain pairs in HiRISE: bright\_dune ↔ dark\_dune and spider ↔ swiss\_cheese.

```
Input patches (64×64 grayscale)
        │
  ┌─────▼──────┐
  │ CNN encoder │  shared weights — two conv blocks → Linear(128) → L2-norm → ℝ⁸
  └─────┬──────┘
        │  eₐ, e_b ∈ S⁷
  ┌─────▼────────────────────────────────────────────┐
  │ Quantum similarity head                          │
  │  Per wire i: RX(π·eₐ[i]) → RY(π·e_b[i])        │
  │  StronglyEntanglingLayers (L=2, 48 params)       │
  │  Local ⟨Z_i⟩ measurements → ℝ⁸                 │
  │  Linear(8,1) + Sigmoid → similarity ∈ (0,1)     │
  └─────┬────────────────────────────────────────────┘
        │
  Triplet loss with hard negative mining
```

**Evaluation metric:** MAP@10 (micro and macro-averaged). Macro-MAP is the primary reported metric — it gives equal weight to all eight terrain classes and is not inflated by the 83.6% "other" majority class.

## Quick start — reproduce the main result

Run all steps in order. Each step depends on the previous one.

```bash
# 1. Create reproducible splits (run once)
python -m scripts.make_splits --data_root data/hirise --output_dir data/splits --seed 42

python -m src.train \

    --checkpoint checkpoints/best_quantum.pt \

```

## Full reproduction — all models and ablations

### Step 1 — Generate splits (one-time)

```bash
python -m scripts.make_splits --data_root data/hirise --output_dir data/splits --seed 42
```

This creates `data/splits/train.json`, `val.json`, and `test.json`. Each file is a JSON object mapping class names to lists of absolute image paths. The split is stratified: every class is split 70/15/15 independently, so minority classes (spider: 476, impact\_ejecta: 231) appear in all three sets.

**Always use the same `--seed` across all training runs** so every model sees the same train/val/test partition.

## Checkpoint format

Each `.pt` file saved by the training scripts contains:

```python
{
    "epoch":          int,            # epoch at which this checkpoint was saved
    "model_state":    OrderedDict,    # model.state_dict()
    "val_map10":      float,          # proper encoder MAP@10 on val split (0 if no split provided)
    "val_triplet_acc": float,         # triplet ranking proxy accuracy
    "n_qubits":       int,
    "n_layers":       int,            # quantum/sliq only
    "q_grad_history": List[float],    # per-epoch mean |∇ PQC weights| (quantum/sliq only)
    "model_type":     str,            # "quantum" | "sliq" | "classical" | "resnet"
    "seed":           int,
}
```

The eval script reads `n_qubits`, `n_layers`, and `model_type` from the checkpoint automatically, so `--model_type` is optional on the eval CLI.

## Evaluation metrics

| Metric | Description |
|--------|-------------|
| **MAP@10 (macro)** | Primary paper metric. AP per class, averaged across all 8 classes. Corrects for 83.6% "other" majority. |
| MAP@10 (micro) | AP per query, averaged across all queries. Reported alongside macro for completeness. |
| Recall@1 | Fraction of queries with ≥1 same-class image in top-1 results. |
| Recall@5 | Fraction of queries with ≥1 same-class image in top-5 results. |
| Precision@K | Average fraction of same-class images among top-K results (K = 1, 5, 10). |

All metrics are computed from **cosine similarity of encoder embeddings**, not the quantum head similarity score. This means the reported MAP@10 measures what the encoder learned, consistent with standard retrieval evaluation.

**Why macro-MAP?** The "other" class has 61,054 images (83.6% of the dataset). Micro-MAP is dominated by "other" — a retrieval system that always returns "other" images performs well on micro-MAP purely due to prior probability. Macro-MAP gives each of the 8 terrain classes equal weight and is the correct metric for evaluating fine-grained terrain discrimination.

## Architecture details

### CNN encoder (`src/encoder.py`)

```
Input:  (B, 1, 64, 64)  grayscale terrain patch
Conv2d(1→32, 3×3, pad=1) + ReLU + MaxPool2d(2)  →  (B, 32, 32, 32)
Conv2d(32→64, 3×3, pad=1) + ReLU + MaxPool2d(2) →  (B, 64, 16, 16)
Flatten                                           →  (B, 16384)
Linear(16384, 128) + ReLU                         →  (B, 128)
Linear(128, N)                                    →  (B, N)
L2-normalize                                      →  (B, N)   on unit sphere S^{N-1}
```

All weights initialized with Kaiming normal. The final L2 normalization ensures encoder outputs `e[i] ∈ [-1, 1]`, so rotation angles `π·e[i] ∈ [-π, π]`.

### Quantum similarity head (`src/quantum_head.py`)

```
Input:  eₐ, e_b ∈ S^{N-1}
Concatenate  →  (B, 2N)
QNode (TorchLayer):
    for i in range(N):
        RX(π·eₐ[i], wire=i)
        RY(π·e_b[i], wire=i)
    StronglyEntanglingLayers(weights, wires=range(N))
    return [⟨Z_i⟩ for i in range(N)]
Linear(N, 1) + Sigmoid
Output: similarity ∈ (0, 1)
```

PQC parameters: `3 × N × L` (48 at N=8, L=2). Post-processing: `N + 1` (9 at N=8). Total quantum head: 57 trainable parameters.

**Implementation note (PennyLane 0.45):** `TorchLayer` passes a `(B, 2N)` tensor to the QNode. Slice with `inputs[..., :N]` (not `inputs[:N]`) to get embedding columns; `inputs[:N]` would slice batch rows.

### Classical head baseline (`baselines/classical_head.py`)

```
Input:  eₐ, e_b ∈ S^{N-1}
Concatenate             →  (B, 2N)
Linear(2N, 64) + ReLU  →  (B, 64)
Linear(64, 1) + Sigmoid →  (B, 1)
Output: similarity ∈ (0, 1)
```

~1,100 parameters at N=8. Larger than the quantum head (57 params) but a more realistic comparison than a single linear layer.

### SLIQ baseline (`baselines/sliq_baseline.py`)

Identical to the quantum head except the encoding layer:

```
# SLIQ encoding (sequential RZ then RX)
for i in range(N):
    RZ(π·eₐ[i], wire=i)    # NOTE: RZ|0⟩ = |0⟩ up to phase — this is a no-op
for i in range(N):
    RX(π·e_b[i], wire=i)
```

vs our encoding:

```
# Ours (interleaved RX then RY — both genuinely rotate from |0⟩)
for i in range(N):
    RX(π·eₐ[i], wire=i)
    RY(π·e_b[i], wire=i)
```

Ablation A2 isolates this axis-choice effect with everything else held constant.

### ResNet50 baseline (`baselines/resnet_baseline.py`)

```
Input:  (B, 1, 64, 64) → repeat → (B, 3, 64, 64)
ResNet50 (pretrained ImageNet, FC removed)  →  (B, 2048, 1, 1)
Flatten                                     →  (B, 2048)
Linear(2048, N) + L2-normalize              →  (B, N)   [encoder]
Classical MLP head (same as above)          →  similarity
```

Uses differential learning rates: `1e-4` for pre-trained backbone, `1e-3` for the randomly-initialized projection and MLP heads.

## Hard negative mining (`src/dataset.py`)

```python
CONFUSABLE_PAIRS = {
    "bright_dune":  "dark_dune",
    "dark_dune":    "bright_dune",
    "spider":       "swiss_cheese",
    "swiss_cheese": "spider",
}
HARD_NEGATIVE_FRACTION = 0.5
```

When constructing a negative sample for anchor class `c`:
- With probability 0.5 (if `c` is in a confusable pair): draw from the partner class.
- With probability 0.5: draw uniformly from any non-anchor class.

This doubles the training signal on the hardest retrievals. Ablation A4 tests this by setting `HARD_NEGATIVE_FRACTION = 0.0`.

## Barren plateau monitoring (`src/train.py`)

Each training epoch, the script tracks `mean |∇ PQC weights|`. If this stays below `1e-5` for 5 consecutive epochs, training automatically switches the quantum layer optimizer from Adam to COBYLA (gradient-free, via `scipy.optimize.minimize`). The classical encoder continues with Adam.

This was not triggered in any reported experiment at N=8, L=2 with local Pauli-Z measurements.

The gradient history is saved in the checkpoint under `q_grad_history` and plotted by `eval.eval` as `eval_outputs/grad_norms_<model>.png`.

## Known issues / limitations

- **Statevector simulation is slow:** At N=8, each forward pass simulates a 256-dimensional statevector. Batch size 16 with two quantum evaluations per triplet = 32 statevector simulations per step. Expect ~5–10 minutes per epoch on CPU.
- **N=12 hard cap:** The training script enforces `n_qubits ≤ 12` (4096-dim statevector). Larger qubit counts require a hardware backend or tensor-network simulator.
- **No GPU quantum simulation:** PennyLane's `default.qubit` with `diff_method="backprop"` runs on CPU. The classical encoder can run on GPU but the quantum head cannot without switching to a hardware backend.
- **Figure placeholders:** `paper/main.tex` still contains `\fbox{}` placeholders for the architecture diagram (Fig. 1) and circuit diagram (Fig. 2). These require manual vector graphics (TikZ / Quantikz).
